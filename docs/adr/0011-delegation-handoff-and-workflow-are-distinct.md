# Delegation, Handoff, and Workflow are distinct

The MVP uses Delegation for multi-employee collaboration: the Root Goal Owner keeps final responsibility, Platform creates bounded Work Items, and assignees return Artifacts for validation. Handoff is a different responsibility-transfer pattern and is intentionally out of MVP scope.

Fixed, high-compliance processes such as refunds, contract signing, permission changes, data deletion, payroll, and formal approvals should be modeled as Platform-controlled Workflows, not free Agent planning. Agents may perform local cognitive steps inside a Workflow, but Platform owns ordering, approvals, state transitions, and final execution.
